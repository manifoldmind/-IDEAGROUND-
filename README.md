# 🌱 IDEAGROUND

> *«Почва, где идеи прорастают в код»*

Это не очередной сборник задач «сдать и забыть».  
`ideaground` — моя личная лаборатория коротких, концептуальных, «голых» заданий.  
Каждая задача — самодостаточный эксперимент, выуженный из брейнштормов, обрезков больших проектов или ночных споров с самим собой.

---

## 🧠 Что здесь живёт?

- **Алгоритмические осколки**  
  *«Реализовать скрытый алгоритм сортировки мусора»*
- **ООП‑прививки**  
  *«Спроектировать иерархию типов для системы лояльности без лишнего жира»*
- **Микро‑ката** (без нишевого жаргона — просто маленькие задачи)  
  *«Написать функцию, которая определяет, можно ли из слова выложить другое слово»*
- **Безумные гипотезы**  
  *«А что, если инкапсуляция через замыкания, а не классы?»*

## 🧭 Как устроен репозиторий
```
ideaground/
├── algo/
│ ├── trash-sort/ # папка задачи
│ │ ├── task.md # условие
│ │ └── solution.py # моя реализация (часто не единственная)
├── oop/
│ ├── loyalty-types/
│ └── composition-vs-inheritance/
├── hybrid/ # задачи на смешанное
└── meta/ # заметки о том, как я придумываю задачи
```

**Принципы:**
- Задача не зависит от других — `cd` внутрь и решай.
- Условие — в `task.md` или прямо в комментариях кода.
- Решение — не истина в последней инстанции. Часто оставляю несколько подходов.

---

## 🎲 Правила игры (для себя и гостей)

1. **Никакой бизнес‑логики**. Только чистая механика, структура, хитрость.
2. **Минимальный код**. Если для демонстрации идеи нужно 200 строк — ты что‑то усложнил(а).
3. **Эксперимент > правильности**. Иногда задача не имеет «верного» решения — только интересные последствия.
4. **Дата создания** — не обязательна. А вот контекст «почему эта задача родилась» — приветствуется.

---

## 🔍 Как это читать / использовать

- **Быстрый просмотр** — листай папки, заголовки задач говорят сами за себя.
- **Решить** — клонируй, выбери задачу, закрой условие и пиши код с нуля.
- **Вдохновиться** — смотри на `task.md` как на вызов своей фантазии: «А как бы я решил(а) это иначе?»

## ☁️ Откуда взялось название

`ideaground` = **idea** + **playground** + **ground** (почва).  
Здесь идеи не просто валяются в песочнице — они пускают корни и из маленьких зёрен вырастают в понятные задачи.
